### Flask Documentation – Pallets Projects

**Quelle:**  
Pallets Projects (2024). *Flask Documentation*.  
https://flask.palletsprojects.com/

**Beitrag zum Projekt:**  
Die Flask-Dokumentation diente als zentrale Referenz für den Aufbau der serverseitigen Webanwendung. Besonders hilfreich waren die Abschnitte zur Definition von Routen, zum Request-Handling sowie zur Verwendung von Sessions für die Nutzerverwaltung. Die Dokumentation zu `redirect`, `url_for` und `flash` unterstützte die Implementierung eines konsistenten Navigations- und Feedbacksystems. Zudem halfen die Beispiele zur Projektstruktur beim Entwurf einer modularen und wartbaren Backend-Architektur.

### SQLAlchemy Documentation – ORM & Datenmodellierung

**Quelle:**  
SQLAlchemy Authors (2024). *SQLAlchemy Documentation*.  
https://docs.sqlalchemy.org/

**Beitrag zum Projekt:**  
Die SQLAlchemy-Dokumentation wurde intensiv für die Modellierung der relationalen Datenbankstruktur genutzt. Besonders relevant waren die Abschnitte zu *Declarative Mapping*, Datentypen und Fremdschlüsselbeziehungen. Diese Inhalte ermöglichten die saubere Abbildung von Nutzern, Onboarding-Daten, Finanzkategorien und Sparzielen. Zusätzlich halfen Beispiele zur Session-Verwaltung dabei, konsistente Datenbanktransaktionen sicherzustellen.

### SQLite Documentation – Persistenzschicht

**Quelle:**  
SQLite Consortium (2024). *SQLite Documentation*.  
https://www.sqlite.org/docs.html

**Beitrag zum Projekt:**  
Die SQLite-Dokumentation unterstützte die Entscheidung für eine leichtgewichtige, dateibasierte Datenbanklösung. Besonders hilfreich waren die Informationen zur Integration von SQLite in kleinere Webanwendungen sowie zu Datentypen und Einschränkungen. Dadurch konnte eine einfache und zuverlässige Persistenz der Daten ohne zusätzliche Serverinfrastruktur realisiert werden.

### Plotly Python – Sankey-Diagramme

**Quelle:**  
Plotly Technologies Inc. (2024). *Plotly Python Documentation – Sankey Diagrams*.  
https://plotly.com/python/sankey-diagram/

**Beitrag zum Projekt:**  
Die Plotly-Dokumentation stellte die Grundlage für die Implementierung des Sankey-Diagramms dar. Besonders hilfreich waren die Codebeispiele zur Definition von Knoten (`nodes`) und Verbindungen (`links`) sowie zur dynamischen Zuweisung von Flusswerten. Diese Inhalte ermöglichten die visuelle Darstellung der Finanzflüsse zwischen Einnahmen, Fixkosten, variablen Ausgaben, Sparbeträgen und Schulden.
